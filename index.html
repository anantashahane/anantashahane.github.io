Evolutionary Strategy
Feature Importance

I designed and implemented a sophisticated solution capable of figuring out feature importance of time series. This innovative approach effectively computes and provides an approximation of the specific weightings assigned to the individual base time series that collectively contribute to the final time series result.

Question: You are given a bunch of time series, it can be completely random and not necessarily sinosuidal waves, and a weighted sum of those time series. How do you solve for the contribution of each of the time series?
Solution: I used an Evolutionary Strategy, that searches for the optimal weightages, and provides (close to) optimal solution for the weightages, and it has been proven to return error less than 10-5 for 40 time series and perfect solution for 10 time series.
Optimisation: I implemented a solution that modulates the search parameter as a function of the error and number of generations since stagnation, giving a really fast and conveerging algorithm, often taking ~400 iterations to find optimal solution when solving for 10 time series.
Philosophy: One can visualise it as a light rain across whole domain of the rough terrain (search parameter proportional to error), then eventually it rains heavily in one of the deep trenches (exploiting local minima), and when it is filled it moves on to other trench (search parameter proportional to stagnation period).

Master's Thesis
A Corridor Model Evolutionary Algorithm for Fast Converging Green Vehicle Routing Problem

Remodelled genetic algorithm for green vehicle routing problem accounting for customers' locality, and engineered corridor model. Reduced average AUC for 92 benchmarks of 500 iterations by 10%.

Question: You have a bunch of customers awaiting to receive their orders, and you are given one depot and a few vehicles. Can you find a tradeoff between cheapest-lowest fuel consumption-and shortest path to serve all the customers?
Solution: I used an NSGA-II (Non-Dominant Sorting Genetic Algorithm-II), for selection process, and introduced Corridor Model from Evolutionary Strategies, into this Discrete Mathematics problem.
Result: This resulted in an algorithm that is n log(n) times slower in execution, but in return converges much faster over the generations, thanks to disciplined repair operators in the mutations.
Philosphy: A random mutation can provide one with optimal solution every once in a while, but choosing carefully early in the optimisation period leads to much more consistent convergence to optimal distance, which turns into extremely good population for exploration of optimal in other objectives.
The research paper is under embargo till June 2024, the repository will be polised and provided after that.

Evolutionary Algorithm
Evolutionary Intelligence

Designed genetic algorithm for searching Deep Learning Architecture for image detection; revealed the link between adjacency matrices and the crossover operator, which improved both convergence speed and stability of the algorithm.

Question: Say you want to perform image detection using deep learning architecture, but there is no know architecture that is discovered to do so; how does one find the desired DL architecture?
Solution: I designed a Genetic Algorithm, that explores various architecture, and is quickly able to converge to an architecture that can perform image detection with 95.05% accuracy.
Optimisation: I discovered a link between the n-point crossover with n-layer architecture, that would significantly improve the performance of the GA in finding the DL Architecture more consistently and more quickly.
Philosphy: A n-point crossover is more likely to share optimal layer found by random mutation than any other crossover, as it is less likely to break a parents' layer into multiple parts, preserving the better performing layers, and potentially introducing newer better layers, improving performance.

Embedded Systems
Sobel on a Chip

Programmed Xilinx FPGA using the Daedalus Framework to execute edge detection using static affine programming paradigm. Used only 66% of the instructed channels in KPN, earning recognition as the top performing software among peers of about 3.75 FPS.

Question: Given a Xilinx FPGA, the goal was to program it to be able to do sobel (edge detection) at as high frame rate as possible.
Process: While working on the assignment, I noticed that the SobelX has a 0 row and sobelY has a 0 column. Hence during the design phase of the Kahn Process Network, I decided to ignore the associated channels, reducing the number of channels required down to 66%.
Results: The expected performance was 2.8-3.5 FPS, but my algorithm provided 3.75 FPS, earning the recognition.
